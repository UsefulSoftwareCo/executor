Verification-viewer bakeoff — 10 prototypes, same data

Every prototype renders the same three runs (a passing browser journey, a passing MCP run, a synthetic failure) from shared mock data. Pick what reads best.

01Filmstrip

Horizontal strip of step screenshots as the primary navigation, like a video-editor timeline; click a frame for full detail.

 02Synced split

Video pinned left, steps right; playback highlights the current step, clicking a step seeks the video.

 03Dense log

Monospace one-line-per-event with aligned columns, expandable in place. CI-output energy, maximum scannability.

 04Timeline

Horizontal time axis; every turn is a positioned bar showing when and how long. Slowness and gaps become visible.

 05Evidence grid

Big card grid of screenshots/video stamped with the assertions they support; transcript tucked behind a toggle.

 06Assertion table

The run as a contract: every assertion in a table (expected/actual/status), failures float up, rows expand to their evidence trail.

 07Storyboard

Comic-strip panels: screenshot + caption + assertion stamps. The whole run on one screen.

 08Inspector

Three-pane IDE layout: turn tree, selected detail, raw JSON. j/k navigation. For debugging, not showcasing.

 09Dashboard-first

The matrix as a health board; cells expand inline to a compact run summary without navigating away.

 10Focused replay

One beat at a time, full screen, arrow keys to advance, pass/fail progress rail. For demoing a run to someone else.
